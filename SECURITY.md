## Reportar una Vulnerabilidad

Si descubres una vulnerabilidad de seguridad en este tema de WordPress, te agradecemos que nos lo reportes de manera responsable.

### Cómo reportar

1. **No** crees un issue público en el repositorio
2. Envía un email a: [tu-email@ejemplo.com] con el asunto "SECURITY: [Descripción breve]"
3. Incluye los siguientes detalles:
   - Descripción de la vulnerabilidad
   - Pasos para reproducir el problema
   - Impacto potencial
   - Sugerencias de solución (si las tienes)

### Qué esperar

- **Respuesta inicial**: Dentro de 48 horas
- **Confirmación**: Dentro de 7 días
- **Resolución**: Dependiendo de la severidad, pero típicamente dentro de 30 días

### Proceso de divulgación

- Trabajaremos contigo para entender y validar la vulnerabilidad
- Te mantendremos informado sobre el progreso
- Una vez resuelta, te daremos crédito (si lo deseas) en el changelog
- Publicaremos un aviso de seguridad si es necesario

## Buenas Prácticas de Seguridad

### Para Desarrolladores

- Mantén WordPress, plugins y el tema actualizados
- No hardcodees credenciales en el código
- Valida y sanitiza todas las entradas de usuario
- Usa funciones de WordPress para escapar datos de salida
- Revisa regularmente los logs de seguridad

### Para Usuarios

- Mantén WordPress y todos los plugins actualizados
- Usa contraseñas fuertes
- Limita los intentos de login
- Usa plugins de seguridad recomendados
- Realiza backups regulares
